코드 작성 규칙

파일 경로 및 링크

  • 유지보수의 편의를 및 HTML 작업자의 퍼포먼스를 위하여 상대 경로로 작업한다. 작업이 완료된 HTML 파일은 로컬 브라우저와 다양한 OS에서 확인이 가능하여야 한다.
  • 웹 문서 전송 속도를 높이기 위하여(Request Count 감소) 최소한의 파일을 사용한다.

캐시정책

  • 캐시정책으로 인하여 리소스 경로(css import 파일 또는 image 경로) 뒤에 고유한 식별자를 붙이도록 한다. 해당경로의 파일이 수정될 경우 해당식별자도 함께 수정하며, 식별자는 날짜+두자리숫자 yyyyMMddNN(예:20220101)를 기본규칙으로 한다.
  • 단, 웹폰트의 경우는 추후 수정되는 파일이 아니며, 속도저하의 요인이 될수 있으므로 캐시정책에서는 제외한다.
/* import 예시 :  import 미사용 */
@import url('page_main.css?20220101');

/* 이미지경로 예시 */
--bgimg1:url('../../img/com/img_set1.png?20220101') no-repeat;

.classname {background:url('파일경로/이미지명.png?20220101') no-repeat 100% 50%;}

/* 웹폰트 선언 제외 */
@font-face {
font-family:'NotoSansKr'; ...
src:url('../font/KR-NotoSansKR-Regular.woff2');
}

코드 들여쓰기

코드의 가독성을 높이고 전체 HTML 구조를 쉽게 파악하기 위하여 들여쓰기 규칙을 준수한다. 마크업의 중첩이 깊어질 때마다 자식 엘리먼트는 1탭 들여 쓰고, 1탭의 크기는 공백 4칸으로 설정한다.

들여쓰기를 하지 않는 경우

  • html의 자식 엘리먼트인 head, body
  • head의 자식 엘리먼트인 meta, link, script
  • body의 자식 첫 번째 엘리먼트 div#container"

빈줄 / 주석

  • 그룹을 감싸고 있는 객체를 구분하기 위하여 코드 그룹 간 1줄씩 빈 줄을 만드는 것은 허용한다. 빈 줄의 간격이 1줄을 초과하지 않으며 개발과 연동되는 그룹의 경우는 빈줄보다 주석을 작성한다.
  • 코드의 가독성을 높이고 전체 HTML 구조를 쉽게 파악하기 위하여 주석사용을 준수한다.
  • 주석 기호와 주석 내용 사이에는 반드시 공백 한 칸이 있어야 한다.
  • 그룹간 1줄 빈줄이 있을경우(예:레이아웃관련)에는 종료주석을 생략할 수 있다.
  • 레이아웃 엘리먼트를 제외한 독립된 콘텐츠 영역의 주석 표기는 선택 사항이다.

기본구조

  • NATIVE 영역은 퍼블리싱 작업에서는 제외(실제 webview 화면 기준임)
  • 페이지 유형에 따라 id값으로 구분한다.

네이밍 규칙

아이디(ID)

  • 카멜 케이스(시작이름은 영문소문자를 사용하되 두번째 단어부터 첫번째 문자를 대문자로 처리)기법을 사용한다.
  • 동일한 이름의 id는 문서에서 한번만 사용이 가능하다.
  • 지정된 레이아웃을 제외하고 UI스타일 지정을 위하여는 id를 사용하지 않는다.
  • 앵커로서 사용되는 엘리먼트는 id 지정이 가능하다.
  • User Interface 동작을 위하여 DOM 선택자로서의 id 지정은 가능하다.
  • 예약어가 있는 경우 예약어를 사용한다.

클래스(CLASS)

  • 네이밍은 하이픈(-) 사용을 기본으로 한다. (클래스 네임이 서버사이드 스크립트의 변수와 자바스크립트의 변수 정의시 혼돈을 방지)
  • 시작의 이름은 영문 소문자를 사용하며 두 번째 단어부터 하이픈으로 연결하며 대문자는 사용하지 않는다.
  • 영문 소문자, 숫자, 하이픈만 사용할 수 있다.
  • 하이픈은 2개 이상의 단어를 조합할 때만 사용하며, 3단계를 초과하여 사용하는 방법은 지양한다.
  • 다중의 클래시 지정을 허용하되 최대 3개 이상 사용하지 않도록 주의한다.

예약어 설명

.none 숨김요소 : 시각, 청각요소 모두 숨김처리대상. 주로 UI 동작관련하여 필요시 사용
.nodata 목록, 검색결과 등 데이터 없을 경우 공통 클래스

이미지(IMAGE)

  • 컨텐츠이미지를 제외한 이미지는 최소화한다(스프라이트기법으로 용량보다는 갯수최소화 지향)
  • 이미지 네이밍은 숫자, 영문소문자, 언더스코어(_)를 조합하여 사용할 수 있다.
  • 이미지 네이밍의 조합은 '형태_의미_순서_상태'의 순서로 조합한다.
  • 같은 이름의 이미지가 두 개 이상 존재하면 숫자로 구분한다.
  • 이미지 네이밍은 이미지의 확장자에 관계 없이 순차적으로 적용한다.
